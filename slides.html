##Customizing slides Slide #1: if you don't know the wi-fi info in advance, just delete/comment out the `<header>` slide. Put all workshop specific images in **framework/img/workshop**. All CSS for this template is written in Sass. Avoid editing the core files (theme-llc.scss, _mixins.scss, _variable.scss). Put all personal customizations in **framework/sass/workshop.scss**. If you'd like to add an improvement or enhancement, please submit a pull request, open an issue or email [christina@ladieslearningcode.com](mailto:christina@ladieslearningcode.com). To change the background image in slide #1 and slide #2, there are several images to choose from in **framework/img**. See **theme-llc.scss**, `.intro` CSS block to make the change. The progress bar at the top of the page can be defined by adding the presentation duration, in minutes, in the `data-duration` attribute on the `body` tag. If you do not want to use it at all, leave the value empty. Since most of the links included in the workshops are to external resources, all links open in a new window by default to allow learners to always have access to the slides. Note that this feature is included using `<base target="_blank">` in the `head` tag. View the rest of the presentation for examples of styles included in the theme. Many styles can be accomplished by adding a specific classes. There is also support for using Markdown as well as including in-slide CSS examples.

Syntax Highlighting

Code highlighting is provided by highlight.js and will be applied to any code wrapped in <pre><code></code></pre>. The syntax highlighting theme files can be found in framework > css > highlightjs-themes. If you wish to change it, update the css link in the <head>. Visit highlight.js to see the full list of supported languages, themes and other usage information.

The default theme for this slide presentation uses a light background because some workshop settings don't allow for the dimming of classroom lights or the projector does not show syntax highlighting on dark backgrounds very well.

For HTML, the left angled bracket, <, needs to be escaped to show. Sometimes, for HTML snippets, the plugin may not register it properly as HTML. If that is the case, add an xml class to the <pre> tag.

Here's a tool to help with the conversion: http://www.htmlescape.net/htmlescape_tool.html

HTML

<!DOCTYPE html>
<html>
  <head>
    <title>Page Title</title>
  </head>
  <body>
    <p class="hello">Paragraph example.</p>
  </body>
</html>

JavaScript

var greeting = "Hello!";

function sayHello() {
  console.log(greeting);
}

sayHello();

body {
  margin: 0;
  padding: 0;
}
##Markdown support Markdown is a text-to-HTML conversion tool and allows you to write using plain text formatting. This slide deck supports automatic conversion from Markdown to HTML. See the [markdown documentation](http://daringfireball.net/projects/markdown/) for formatting syntax. Markdown support can be added in two ways. See the next two slides for more info.
##Markdown: Option 1 Include the `data-markdown` attribute in the section tag. ```xml <section class="slide" data-markdown> ##This is markdown! </section> ``` **If you need to use any HTML tags in the slide or add a class/id to an element, the entire slide has to be written in HTML and the markdown syntax will no longer be supported. See option 2 for more info about mixing markdown and HTML.** Code blocks can be denoted either by three back ticks (```) **or** with one 4 space tab. Using back ticks, immediately followed by the language, will allow you to specify a language for syntax highlighting. View the HTML source or see below for examples. #### Javascript Example: \```javascript ```javascript var myJSVariable = "Syntax Highlighting for JavaScript"; ``` #### CSS Example \```css ```css .css-selector { property: value; } ``` #### HTML Example For HTML, set `xml` as the language to get nicer syntax highlighting. Also, just like the `<code>` in the previous example, the left angled bracket of the HTML tag needs to be escaped. View source to see example. \```xml ```xml <header> <h1>I'm an H1!</h1> <p>This is a paragraph</p> </header> ``` #### indented example code block that has to guess at the language and likely gets it wrong.

Markdown: Option 2

Add the data-markdown attribute to the <section> tag and wrap the content in a <script type="text/template"></script>. With this option, you can mix HTML markup (does not need to be escaped) with Markdown syntax though avoid using any additional <script> tags within.

Also, using single or double quotes in the content may throw off the syntax highlighting in your editor, though the content will still be rendered properly in the browser. You can use the HTML code instead.

<section class="slide" data-markdown>
  <script type="text/template">
    ## Heading2

    A paragraph with some text and a [link](http://hakim.se).

    <p class="delayed">delayed text</p>
  </script>
</section>

You can also add an attribute like this:

Footnote text<!-- .element: class="footnote" -->

Be sure to place the snippet before any nested elements.

Footnote text and <!-- .element: class="footnote" --> [link](http://website.com)

Will be converted to:

<p class="footnote">A paragraph with some text and a <a href="http://hakim.se">link</a>.</p>
